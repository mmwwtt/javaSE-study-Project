Redis:非关系型键值对数据库
	键类型：字符串
	值类型：字符串，列表，集合，散列表，有序集合
	Redis数据存在内存中，读写速度非常快，经常用来做分布式锁
	支持事务，持久化等
	使用场景：缓存，计算器，分布式锁
	优点：
		读写快
		支持数据持久化
		支持事务，所有操作都是原子性的，还支持几个操作合并的原子性
		数据结构丰富
		支持主从复制，主机自动将数据同步到从机，可读写分离
	缺点：
		数据库容量受物理内存限制，不能用作对海量数据的高性能读写
		不具备自动容错和恢复功能，主从机死机会使前端读写请求失败，需要重启机器
	设置值
		set key value [ex seconds] [px millseconds] [nx|xx]
		ex seconds：为键设置秒级过期时间，跟 setex 效果一样
		px millseconds：为键设置毫秒级过期时间
		nx：键必须不存在才可以设置成功，用于添加，跟 setnx 效果一样。由于 Redis 的单线程命令处理机制，如果多个客户端同时执行，则只有一个客户端能设置成功，可以用作分布式锁的一种实现。
		xx：键必须存在才可以设置成功，用于更新
	获取值
	get key ，如果不存在返回 nil
	批量设置值
	mset key value [key value…]
	批量获取值
	mget key [key…]
